Execute JavaScript or TypeScript code in a sandboxed REPL environment.

## Language Support

Supports both JavaScript and TypeScript:
- **Auto-detection**: TypeScript syntax is automatically detected and transpiled
- **Explicit**: Use `language: "typescript"` to force TypeScript mode

**TypeScript Example:**
```typescript
interface User {
  name: string;
  age: number;
}

const users: User[] = [
  { name: "Alice", age: 30 },
  { name: "Bob", age: 25 }
];

const avgAge: number = users.reduce((sum, u) => sum + u.age, 0) / users.length;
console.log(`Average age: ${avgAge}`);
```

## Security Features

- **Isolated execution**: Code runs in a VM sandbox with no access to Node.js internals
- **No eval/Function**: Dynamic code execution is blocked
- **No require/import**: Module loading is blocked
- **No process/global**: System access is blocked
- **Timeout enforcement**: Default 30s, max 5 minutes

## State Persistence

Variables persist automatically within the same agent chat session. No manual session management needed.

**Example (persistent within session):**
```javascript
// First execution
const data = [1, 2, 3, 4, 5];
const sum = data.reduce((a, b) => a + b, 0);
console.log(`Sum: ${sum}`);

// Second execution (same chat session)
const average = sum / data.length;  // Variables from first execution persist
console.log(`Average: ${average}`);
```

## Context Loading

Use the `context` parameter to preload data:

```javascript
// With context: {"users": [{"name": "Alice"}, {"name": "Bob"}]}
for (const user of context.users) {
  console.log(user.name);
}
```

## Async Code (CRITICAL)

The `code` parameter MUST always be **valid JavaScript code**, never raw JSON.
Top-level `await` is **NOT supported** — it throws `SyntaxError`.

When calling `tools.*` methods, you **MUST** wrap code in an async IIFE:

```javascript
(async () => {
  // your async code here
})()
```

**Example — Read a file:**
```javascript
// Read src/config.ts
(async () => {
  const content = await tools.readFile('src/config.ts');
  console.log(content);
})()
```

**Example — Curate to context tree:**
```javascript
// Curate alert engine to design/alert_engine
(async () => {
  const result = await tools.curate([{
    type: 'UPSERT',
    path: 'design/alert_engine',
    title: 'Alert Engine',
    reason: 'Document alert engine architecture',
    content: {
      rawConcept: {
        task: 'Document the Alert Engine component',
        flow: 'Events -> Alert Router -> Deduplication -> Routing'
      },
      narrative: {
        structure: 'The Alert Engine consumes events and routes alerts.',
        highlights: 'Deduplication, escalation, SLA tracking',
        rules: 'Critical: PagerDuty + Slack. High: Slack + Email.'
      }
    },
    domainContext: {
      purpose: 'System architecture and design documentation',
      scope: { included: ['Component architecture', 'Design patterns'] }
    },
    topicContext: {
      overview: 'Alert Engine architecture and routing rules',
      keyConcepts: ['Alert routing', 'Deduplication', 'SLAs']
    }
  }]);
  console.log(result);
})()
```

**Example — Search knowledge:**
```javascript
// Search for authentication patterns
(async () => {
  const results = await tools.searchKnowledge('authentication');
  console.log(results);
})()
```

**Sync code does NOT need a wrapper:**
```javascript
// Transform user data
const users = [{name: 'Alice'}, {name: 'Bob'}];
const names = lodash.map(users, 'name');
console.log(names);
```

**Wrong patterns (will fail):**
```javascript
// WRONG: Raw JSON as code — code must be JavaScript!
{"type": "UPSERT", "path": "design/auth", ...}

// WRONG: Top-level await
const file = await tools.readFile('f.ts');

// WRONG: import/require
import { readFile } from 'tools';
```

## Available `tools.*` Methods

All methods are async and require the async IIFE wrapper.

- **tools.readFile(path, options?)** — Read file contents
- **tools.writeFile(path, content, options?)** — Write to a file
- **tools.glob(pattern, options?)** — Find files by glob pattern
- **tools.grep(pattern, options?)** — Search file contents by regex
- **tools.listDirectory(path?, options?)** — List directory tree
- **tools.curate(operations[], options?)** — Curate knowledge (ADD/UPDATE/UPSERT/MERGE/DELETE)
- **tools.searchKnowledge(query, options?)** — Search curated knowledge
- **tools.detectDomains(domains[])** — Detect domains from text

## Available Globals

- **Console**: console.log, console.error, console.warn, console.info
- **Core Types**: Object, Array, String, Number, Boolean, Map, Set, Date, RegExp, Promise
- **Math/JSON**: Math, JSON, Intl
- **Utilities**: parseInt, parseFloat, isNaN, isFinite, encodeURI, decodeURI

## Available Packages

Pre-loaded utility packages for computation (no file system access):

**Data Manipulation:**
- **lodash** - Utilities (`lodash.map()`, `lodash.groupBy()`, `lodash.sortBy()`)
- **ramda** - Functional programming

**String & Regex:**
- **changeCase** - Case conversion (`changeCase.camelCase('hello world')`)
- **pluralize** - Pluralization (`pluralize('word')`)
- **slugify** - URL slugs (`slugify('Hello World')`)

**Pattern Matching (in-memory):**
- **minimatch** - Glob matching for strings (`minimatch('file.ts', '*.ts')`)
- **micromatch** - Advanced glob matching

**Data Validation:**
- **zod** - Schema validation
- **validator** - String validation (`validator.isEmail('test@example.com')`)
- **json5** / **yaml** - Data parsing

**Date/Time:**
- **dateFns** / **dayjs** - Date manipulation

**Math:**
- **mathjs** - Advanced math
- **decimalJs** - Precision decimals

**ID Generation:**
- **uuid** / **nanoid** - ID generation

**Example:**
```javascript
// Pattern matching on strings (not files)
const files = ['src/index.ts', 'src/utils.js', 'README.md'];
const tsFiles = files.filter(f => minimatch(f, '**/*.ts'));

// Data transformation
const users = [{name: 'Alice', role: 'admin'}, {name: 'Bob', role: 'user'}];
const byRole = lodash.groupBy(users, 'role');
```

**Note:** For file operations, use existing tools (read_file, glob_files, grep_content).

## Output

- `stdout`: Console output (console.log, console.info)
- `stderr`: Error output (console.error, console.warn, exceptions)
- `locals`: Current context state (serialized variables)
- `returnValue`: Value of the last expression
- `executionTime`: Execution time in milliseconds

## When to Use

- Data analysis and transformation
- JSON processing and manipulation
- Algorithm prototyping with type safety
- Complex calculations
- Array/object operations
