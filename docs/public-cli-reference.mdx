ByteRover CLI (`brv`) is a command-line tool for managing your AI development workflow with ByteRover. Use it to authenticate, retrieve memories from your knowledge base, manage projects and spaces, and maintain your context tree.

For help with any command, run `brv --help` or `brv <command> --help`.

## Getting Started

Before using ByteRover CLI, you need to authenticate and initialize your project:

```bash
# First, authenticate with ByteRover
brv login

# Then, initialize your project (select team and space)
brv init
```

Authentication uses OAuth 2.0 with PKCE flow. Your credentials are securely stored in your system's keychain. The `brv init` command creates a `.brv/` directory in your project with configuration and context tree structure.

## CLI Commands

| Command                                         | Description                                  | Example                                            |
| ----------------------------------------------- | -------------------------------------------- | -------------------------------------------------- |
| `brv login`                                     | Authenticate with ByteRover using OAuth 2.0  | `brv login`                                        |
| `brv logout`                                    | Log out and clear authentication             | `brv logout --yes`                                 |
| `brv status [DIRECTORY]`                        | Show CLI status and project information      | `brv status`                                       |
| `brv init`                                      | Initialize a project with ByteRover          | `brv init`                                         |
| `brv space list`                                | List all spaces for the current team         | `brv space list --all`                             |
| `brv space switch`                              | Switch to a different team or space          | `brv space switch`                                 |
| `brv push`                                      | Push context tree to ByteRover memory storage | `brv push`                                         |
| `brv pull`                                      | Pull context tree from ByteRover memory storage | `brv pull`                                       |
| `brv query <QUERY>`                             | Query information from the context tree      | `brv query "How is auth implemented?"`             |
| `brv curate [CONTEXT]`                          | Add content to context tree                  | `brv curate "JWT tokens expire in 24h"`            |
| `brv clear [DIRECTORY]`                         | Reset context tree to default domains        | `brv clear`                                        |
| `brv gen-rules`                                 | Generate rule instructions for coding agents | `brv gen-rules --agent Claude`                     |

## Command Details

### `brv login`

Authenticate with ByteRover using OAuth 2.0 with PKCE flow. Opens your browser for secure authentication and stores tokens in your system's keychain.

**Arguments**

None

**Flags**

None

**Examples**

```bash
# Authenticate with ByteRover
brv login

# Re-authenticate after token expires
brv login
```

**Notes**

- First command you should run when setting up ByteRover
- Opens your default browser for authentication
- Tokens are stored securely in system keychain (macOS Keychain, Windows Credential Manager, Linux libsecret)
- If authentication expires, simply run `brv login` again

---

### `brv logout`

Log out of ByteRover CLI and clear authentication. Removes credentials from system keychain while preserving all local project files.

**Arguments**

None

**Flags**

| Flag        | Description              | Default |
| ----------- | ------------------------ | ------- |
| `-y, --yes` | Skip confirmation prompt | `false` |

**Examples**

```bash
# Log out (with confirmation)
brv logout

# Log out without confirmation
brv logout --yes
brv logout -y
```

**Notes**

- Clears authentication token from system keychain
- Does NOT affect local project files (`.brv/` directory is preserved)
- Shows confirmation prompt with current user email unless `--yes` flag is used
- You can run `brv login` again to re-authenticate

---

### `brv status`

Show CLI status and project information. Displays local context tree managed by ByteRover CLI.

**Arguments**

| Argument    | Type   | Required | Description                                       |
| ----------- | ------ | -------- | ------------------------------------------------- |
| `DIRECTORY` | string | No       | Project directory (defaults to current directory) |

**Flags**

| Flag                    | Description                      | Default |
| ----------------------- | -------------------------------- | ------- |
| `-f, --format <option>` | Output format: `table` or `json` | `table` |

**Examples**

```bash
# Show status for current directory
brv status

# Show status for a specific directory
brv status /path/to/project

# Output status as JSON
brv status --format json
```

**Output**

The status command displays:

- CLI version
- Authentication status (logged in user or "Not logged in")
- Current directory
- Project initialization status (team and space info if initialized)
- Context tree changes (git-style diff showing modified/added/deleted files)

---

### `brv init`

Initialize a project with ByteRover. Creates `.brv/config.json` with team and space selection, and initializes the context tree structure.

**Arguments**

None

**Flags**

| Flag              | Description                                             | Default |
| ----------------- | ------------------------------------------------------- | ------- |
| `-f, --force`     | Force re-initialization without confirmation prompt     | `false` |

**Examples**

```bash
# Initialize project (interactive team/space selection)
brv init

# Complete first-time setup
brv login
brv init

# Force re-initialization without confirmation (for scripts/automation)
brv init --force
```

**Re-initialization**

If the project is already initialized, `brv init` will prompt you to confirm re-initialization. This allows you to switch to a different team/space or reset your project.

**Re-initializing will:**
- Remove the entire `.brv/` directory and all its contents
- Allow you to select a new team/space
- Create a fresh configuration and context tree
- Regenerate rule instructions

**To re-initialize:**
1. Run `brv init` in an initialized project
2. Review the warning about what will be deleted
3. Confirm to proceed or cancel to keep current configuration
4. Use `--force` flag to skip confirmation (useful for automation)

**Notes**

- Requires `brv login` first
- Interactive prompts guide you through team and space selection
- Creates `.brv/` directory with configuration and context tree structure
- Automatically runs `brv gen-rules` after initialization
- **Warning:** Re-initialization deletes all local context tree data
- To switch spaces without clearing data, use `brv space switch` instead

### `brv space list`

List all spaces for the current team. Requires project initialization to determine which team to query.

**Arguments**

None

**Flags**

| Flag                   | Description                                    | Default |
| ---------------------- | ---------------------------------------------- | ------- |
| `-a, --all`            | Fetch all spaces (may be slow for large teams) | `false` |
| `-j, --json`           | Output in JSON format                          | `false` |
| `-l, --limit <value>`  | Maximum number of spaces to fetch              | `50`    |
| `-o, --offset <value>` | Number of spaces to skip                       | `0`     |

**Examples**

```bash
# List first 50 spaces (default)
brv space list

# List all spaces
brv space list --all

# Manual pagination: get 10 spaces at a time
brv space list --limit 10
brv space list --limit 10 --offset 10

# Output as JSON
brv space list --json
```

**Notes**

- Requires `brv init` first (reads team ID from config)
- Default limit is 50 spaces
- Use `--all` to fetch all spaces (internally paginates at 100 items per page)
- Use `--limit` and `--offset` for manual pagination control

### `brv space switch`

Switch to a different team or space. Updates `.brv/config.json` with new selection.

**Arguments**

None

**Flags**

None

**Examples**

```bash
# Switch team or space (interactive)
brv space switch
```

**Notes**

- Requires `brv init` first (project must be initialized)
- Displays current team and space configuration
- Interactive prompts guide you through new team and space selection
- Updates configuration but does NOT initialize context tree

---

### `brv push`

Push context tree to ByteRover memory storage. Uploads your accumulated knowledge to the cloud.

**Arguments**

None

**Flags**

| Flag                   | Description                            | Default |
| ---------------------- | -------------------------------------- | ------- |
| `-b, --branch <value>` | ByteRover branch name (not Git branch) | `main`  |
| `-y, --yes`            | Skip confirmation prompt               | `false` |

**Examples**

```bash
# Push to default branch (with confirmation)
brv push

# Push to a specific branch
brv push --branch develop
brv push -b feature-auth

# Skip confirmation prompt
brv push --yes
brv push -y
```

**Notes**

- Requires `brv login` and `brv init` first
- Context tree must exist to push
- Branch concept is ByteRover-internal, NOT related to Git branches
- Shows confirmation dialog unless `--yes` flag is used
- Snapshots the current context tree state for version tracking

---

### `brv pull`

Pull context tree from ByteRover memory storage. Downloads the latest knowledge from the cloud to your local context tree.

**Arguments**

None

**Flags**

| Flag                   | Description                            | Default |
| ---------------------- | -------------------------------------- | ------- |
| `-b, --branch <value>` | ByteRover branch name (not Git branch) | `main`  |

**Examples**

```bash
# Pull from default branch
brv pull

# Pull from a specific branch
brv pull --branch develop
brv pull -b feature-auth
```

**Notes**

- Requires `brv login` and `brv init` first
- Checks for local changes before pulling (push local changes first if any)
- Syncs files to local context tree
- Branch concept is ByteRover-internal, NOT related to Git branches

---

### `brv query`

Query and retrieve information from the context tree using natural language.

**Arguments**

| Argument | Type   | Required | Description                                       |
| -------- | ------ | -------- | ------------------------------------------------- |
| `QUERY`  | string | Yes      | Natural language question about your project      |

**Flags**

None

**Examples**

```bash
# Query about implementation details
brv query "How is user authentication implemented?"
brv query "What are the API rate limits and where are they enforced?"

# Query about project structure
brv query "What testing strategies are used?"
```

**Notes**

- Context tree must exist (run `brv init` first)
- Uses AI to search and analyze your context tree
- Best results with specific, detailed questions
- Avoid vague queries like "auth" or "show me code"

---

### `brv curate`

Add content to the context tree using interactive or autonomous mode.

**Arguments**

| Argument  | Type   | Required | Description                                            |
| --------- | ------ | -------- | ------------------------------------------------------ |
| `CONTEXT` | string | No       | Content to add (triggers autonomous mode if provided) |

**Flags**

None

**Examples**

```bash
# Interactive mode (manually choose domain/topic)
brv curate

# Autonomous mode with internal LLM (default)
brv curate "User authentication uses JWT tokens with 24h expiry"
brv curate "React components follow atomic design pattern"
```

**Notes**

- Context tree must exist (run `brv init` first)
- **Interactive mode**: Navigate context tree, select domain/topic, creates context.md file
- **Autonomous mode**: AI agent automatically determines best location and adds content

### `brv clear`

Reset the context tree to its original state with 6 default domains. Removes all custom domains and content, restoring the initial configuration created by `brv init`.

**Default Domains Restored**

- `code_style` - Style guidelines and quality standards
- `design` - UI libraries, themes, and design guidelines
- `structure` - Project structure, components, and context
- `compliance` - Security and compliance information
- `testing` - Testing implementation context
- `bug_fixes` - Bug fixing logic and procedures

**Arguments**

| Argument    | Type   | Required | Description                                       |
| ----------- | ------ | -------- | ------------------------------------------------- |
| `DIRECTORY` | string | No       | Project directory (defaults to current directory) |

**Flags**

| Flag        | Description              | Default |
| ----------- | ------------------------ | ------- |
| `-y, --yes` | Skip confirmation prompt | `false` |

**Examples**

```bash
# Reset context tree (with confirmation)
brv clear

# Reset without confirmation
brv clear --yes
brv clear -y

# Reset context tree in specific directory
brv clear /path/to/project
```

**Notes**

- Context tree must exist (created by `brv init`)
- Shows confirmation prompt unless `--yes` flag is used
- This action removes all custom domains and content
- Cannot be undone - all custom context will be lost
- Does not affect `.brv/config.json` or project initialization

---

### `brv gen-rules`

Generate rule instructions for coding agents to work with ByteRover correctly. Creates agent-specific rule files with instructions for using ByteRover CLI in their workflow.

**Arguments**

None

**Flags**

| Flag                  | Description                                      | Default |
| --------------------- | ------------------------------------------------ | ------- |
| `-a, --agent <value>` | Agent to generate rules for (skips interactive prompt) | None    |

**Examples**

```bash
# Generate rules (interactive agent selection)
brv gen-rules

# Generate rules for a specific agent
brv gen-rules --agent Claude
brv gen-rules -a Cursor
```

**Notes**

- Interactive search prompt for agent selection (if `--agent` flag not provided)
- Prompts for overwrite confirmation if rule file already exists
- Automatically called by `brv init` command
- Generated rules help coding agents understand ByteRover workflow
- Rules include commands, workflows, and best practices for the selected agent

## File Structure

ByteRover CLI creates the following directory structure in your project:

```
.brv/
├── config.json                # Project configuration (team/space)
├── context-tree/              # Your knowledge base
│   ├── code_style/            # Style guidelines and quality standards
│   ├── design/                # UI libraries and design guidelines
│   ├── structure/             # Project structure and components
│   ├── compliance/            # Security and compliance
│   ├── testing/               # Testing implementation context
│   └── bug_fixes/             # Bug fixing logic and procedures
└── snapshots/                 # Version snapshots (git-style)
    └── latest.json            # Latest context tree snapshot
```

**Key Files and Directories**

- **config.json**: Stores your team ID, space ID, and project metadata
- **context-tree/**: Hierarchical structure organizing project knowledge by domains and topics
- **snapshots/**: Version control snapshots tracking context tree changes over time
- Each domain can contain multiple topics, and each topic has a `context.md` file with actual content
